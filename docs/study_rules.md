# Study Rules

## 1. 기본 규칙

1. main 브랜치 직접 push 금지
2. 반드시 브랜치에서 작업 후 Pull Request 생성
3. 최소 1명 리뷰 후 merge
4. 문제 링크와 문제 유형 작성
5. 상대방 코드를 읽고 질문 하나 이상 남기기

---

## 2. 폴더 규칙

- 사람별 폴더 사용
- 주차별 폴더 사용
- 파일명은 문제 플랫폼 + 문제번호 기준

예시

- `changoh/week01/BOJ_1759.java`
- `friend/week02/BOJ_9663.java`

---

## 3. PR 규칙

PR 본문에 아래 4가지는 꼭 작성합니다.

- 문제 링크
- 사용 알고리즘
- 풀이 아이디어
- 리뷰 받고 싶은 부분

---

## 4. 리뷰 규칙

좋은 리뷰 예시

- 이 부분은 visited 없이도 가능한 것 같습니다.
- 여기서 Set을 쓰면 중복 처리가 더 쉬울 것 같습니다.
- 시간복잡도가 O(N^2)인데 입력 크기에서 괜찮은지 확인해보면 좋겠습니다.
- 메서드를 분리하면 가독성이 더 좋아질 것 같습니다.

피해야 할 리뷰

- 그냥 이상함
- 너무 별로임
- 다시 짜세요

---

## 5. 추천 진행 방식

- 각자 먼저 풀이
- PR 생성
- 상대방 풀이와 비교
- 더 짧은 풀이 / 더 직관적인 풀이 고민
- 스터디 끝나고 공통 회고 정리

---

## 6. 회고 포인트

스터디 끝나고 아래를 짧게 정리하면 좋습니다.

- 오늘 새로 배운 알고리즘 포인트
- 내가 자주 실수하는 패턴
- 상대 코드에서 배운 점
- 다음에 다시 풀 문제
